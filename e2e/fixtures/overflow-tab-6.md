# Overflow Tab 6

## Overview

This is a test document for end-to-end testing. It contains multiple paragraphs with enough text to select and comment on.

## Section One

The authentication system supports email and password login. Users must provide valid credentials to access the application. The system validates all inputs before processing.

## Section Two

Rate limiting prevents brute force attacks. Failed login attempts are tracked per IP address. After five failures the account is temporarily locked.

## Section Three

All passwords are hashed with bcrypt. Session tokens are stored in HTTP-only cookies. CSRF protection uses the double-submit cookie pattern.
